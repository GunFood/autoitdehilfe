_WinAPI_ChildWindowFromPointEx

Beschreibung anzeigen in

Ermittelt, ob und welche der zum angegebenen übergeordneten Fenster gehörenden untergeordneten Fenster den angegebenen Punkt enthalten

#include <WinAPISysWin.au3>
_WinAPI_ChildWindowFromPointEx ( $hWnd, $tPOINT [, $iFlags = 0] )

Parameter

$hWnd Handle zum übergeordneten Fenster..
$tPOINT $tagPOINT-Struktur, die die Client-Koordinaten (relativ zu hwndParent) des zu prüfenden Punktes definiert.
$iFlags [optional] Die Flags, die angeben, welche untergeordneten Fenster übersprungen werden sollen. Dieser Parameter kann einen oder mehrere der folgenden Werte annehmen:
    $CWP_ALL (Standard)
    $CWP_SKIPINVISIBLE
    $CWP_SKIPDISABLED
    $CWP_SKIPTRANSPARENT

Rückgabewert

Erfolg: Handle zum ersten untergeordneten Fenster. Befindet sich der Punkt innerhalb des übergeordneten Fensters, aber in keinem untergeordneten Fenster, das die Kriterien erfüllt, ist der Rückgabewert ein Handle zum übergeordneten Fenster. Liegt der Punkt außerhalb des übergeordneten Fensters, ist der Rückgabewert 0.
Fehler: 0.

Bemerkungen

Das System verwaltet eine interne Liste, die die Handles der mit einem übergeordneten Fenster verbundenen untergeordneten Fenster enthält.
Die Reihenfolge der Handles in der Liste hängt von der Z-Reihenfolge der untergeordneten Fenster ab.
Wenn mehr als ein untergeordnetes Fenster den angegebenen Punkt enthält, wird ein Handle auf das erste Fenster in der Liste zurückgegeben, das den Punkt enthält und die durch $iFlags angegebenen Kriterien erfüllt.
