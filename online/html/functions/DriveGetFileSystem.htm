DriveGetFileSystem



Beschreibung anzeigen in

Gibt den Typ des Dateisystems eines Laufwerks (bzw. Partition) zurück.

DriveGetFileSystem ( "path" )

Parameter

path Laufwerkspfad, dessen Information abgefragt wird.

Rückgabewert

Erfolg: Der Dateisystemtyp des Laufwerks als String (siehe Tabelle unten).
Fehler: Setzt @error auf 1.

Return Value Interpretation
$DT_UNDEFINED (1) (numerisch) Laufwerk enthält kein Medium(CD, Floppy, Zip) oder das Medium ist unformatiert (RAW).
$DT_FAT ("FAT") Typisches Dateisystem für Laufwerke unter ~500 MB, wie z.B. Floppy, RAM-Disks, USB-Sticks, etc.
$DT_FAT32 ("FAT32") Typisches Dateisystem für Windows 9x/Me Festplatten.
$DT_EXFAT ("exFAT") Typisches Dateisystem für Laufwerke mit Dateien über 4 GB wie z. B. RAM-Disks, USB-Sticks, etc.
$DT_NTFS ("NTFS") Typisches Dateisystem für Windows Festplatten.
$DT_UNWFS ("NWFS") Typisches Dateisystem für Novell Netware Fileserver.
$DT_CDFS ("CDFS") Kennzeichnet typischerweise eine CD (oder ein als ISO-Image gemountetes, virtuelles CD-Laufwerk).
$DT_UDF ("UDF") Kennzeichnet typischerweise eine DVD.

Die Konstanten sind in "AutoItConstants.au3" definiert.

Bemerkungen

Die Liste der möglichen Rückgabewerte kann unvollständig sein.

Verwandte Funktionen

DriveGetDrive, DriveGetLabel, DriveGetSerial, DriveGetType, DriveSetLabel, DriveSpaceFree, DriveSpaceTotal, DriveStatus

Beispiel

#include <MsgBoxConstants.au3>

Local $sFileSystem = DriveGetFileSystem(@HomeDrive & "\") ; Findet den Dateisystem Typ des Hauptverzeichnisses des aktuellen Benutzers, normalerweise ist dies C:\.
MsgBox($MB_SYSTEMMODAL, "Dateisystem Typ:", $sFileSystem)
