_WinAPI_DuplicateEncryptionInfoFile

Kopiert die EFS Metadaten einer Datei oder eines Verzeichnisses in eine andere Datei oder Verzeichnis

#include <WinAPIFiles.au3>
_WinAPI_DuplicateEncryptionInfoFile ( $sSrcFilePath, $sDestFilePath [, $iCreation = 2 [, $iAttributes = 0 [, $tSecurity = 0]]] )

Parameter

$sSrcFilePath Der Name der Datei oder des Verzeichnisses dessen EFS Metadaten kopiert werden sollen.
Die Quelldatei oder das Quellverzeichnis müssen verschlüsselt sein.
$sDestFilePath Der Name der Datei oder des Verzeichnisses wohin die EFS Metadaten kopiert werden sollen.
Die Zieldatei oder das Zielverzeichnis müssen vor dem Aufruf dieser Funktion nicht verschlüsselt sein.
Wenn die Quelle eine Datei ist, so muss das Ziel ebenfalls eine Datei sein.
Selbiges gilt analog für Verzeichnisse.
$iCreation [optional] Gibt an wie die Zieldatei oder das Zielverzeichnis geöffnet werden sollen.
Die folgenden Werte sind für diesen Parameter gültig:
    $CREATE_NEW
    $CREATE_ALWAYS (Standard)
$iAttributes [optional] Die Datei- oder Verzeichnissattribute der Zieldatei oder des Zielverzeichnisses.
Das $FILE_READ_ONLY wird momentan von dieser Funktion nicht verarbeitet.
$tSecurity [optional] Die $tagSECURITY_ATTRIBUTES Struktur gibt die Sicherheitsattribute der Zieldatei oder des Zielverzeichnisses an sofern es nicht bereits existiert.
Wenn dieser Parameter 0 ist (Standard), dann bekommt die Datei oder das Verzeichnis einen Standardsicherheitsdeskriptor.
Die Zugangskontrollliste (access-control list (ACL)) im Standardsicherheitsdeskriptor für eine Datei oder ein Verzeichnis werden vom übergeordneten Verzeichnis geerbt.

Rückgabewert

Erfolg: 1.
Fehler: 0 und setzt das @error Flag auf ungleich null. @extended kann den Systemfehlercode enthalten.

Bemerkungen

Der Aufrufer sollte den EFS-Schlüssel für die Quelldatei oder das Quellverzeichnis besitzen oder wenigstens wenigstens die $READ_ATTRIBUTE Zugangskontrollliste (ACL) für die Quelldatei oder das Quellverzeichnis.
Die angegebene Quell-/Zieldateien und Quell-/Zielverzeichnisses sollten sich auf dem selben Computer befinden, anderenfalls schlägt diese Funktion fehl.
